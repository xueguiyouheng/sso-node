# System B 系统

这是一个混合型应用程序，使用Next.js构建，包含公共内容和受保护内容。

## 功能特点

- 混合应用程序 - 同时提供公共和受保护内容
- 与SSO系统集成，支持单点登录
- 未认证用户可以访问公共内容
- 认证用户可以访问额外的受保护内容
- 响应式设计，适配各种设备
- 个人资料管理功能

## 使用技术

- Next.js
- React
- Axios用于HTTP请求
- CSS3样式

## 页面功能

- 公共页面：未认证用户可以访问的内容
- 受保护页面：认证用户可以访问的内容
- 登录/登出功能：与SSO系统集成
- 个人资料管理：允许认证用户更新个人信息

## 环境要求

- Node.js (版本14或更高)
- SSO服务器正在运行
- 登录系统正在运行

## 启动系统

```bash
npm install
npm run dev
```

系统将在端口3003上运行，访问地址：http://localhost:3003

## 工作流程

1. 用户访问System B (http://localhost:3003)
2. 未认证用户可以查看公共内容
3. 用户点击登录按钮进行认证
4. 认证成功后可以访问受保护内容
5. 用户可以点击登出按钮清除会话

## SSO集成原理

- 使用HTTP-only cookies存储会话令牌
- 通过重写规则将API请求转发到SSO服务器
- 实现集中式认证管理
- 支持跨系统单点登录和登出

## 个人资料管理

System B现在支持个人资料管理功能，允许认证用户：

- 更新个人信息（姓名、邮箱、电话等）
- 更改登录密码
- 添加个人简介和头像

认证用户可以通过侧边栏中的"个人资料"链接或主页上的个人资料管理部分访问个人资料管理页面。
