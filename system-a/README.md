# System A 系统

这是一个完全受保护的应用程序，使用React和Vite构建，需要用户认证才能访问任何内容。

## 功能特点

- 完全受保护的应用程序 - 所有内容都需要身份验证
- 与SSO系统集成，支持单点登录
- 登录按钮重定向到SSO登录页面
- 登出按钮清除会话并重定向到登录页面
- 响应式设计，适配各种设备
- 个人资料管理功能

## 使用技术

- React
- Vite
- Axios用于HTTP请求
- CSS3样式

## 页面功能

- 公共页面：未认证用户无法访问，直接重定向到登录页面
- 受保护页面：认证用户可以访问的内容
- 登录/登出功能：与SSO系统集成
- 个人资料管理：允许用户更新个人信息和更改密码

## 环境要求

- Node.js (版本14或更高)
- SSO服务器正在运行
- 登录系统正在运行

## 启动系统

```bash
npm install
npm run dev
```

系统将在端口3000上运行，访问地址：http://localhost:3000

## 工作流程

1. 用户访问System A (http://localhost:3000)
2. 系统检测认证状态，未认证用户重定向到登录页面
3. 用户在登录页面认证后重定向回System A
4. 系统验证令牌并显示受保护内容
5. 用户可以点击登出按钮清除会话

## SSO集成原理

- 使用HTTP-only cookies存储会话令牌
- 通过代理配置将API请求转发到SSO服务器
- 实现集中式认证管理
- 支持跨系统单点登录和登出

## 个人资料管理

System A现在支持个人资料管理功能，允许用户：

- 更新个人信息（姓名、邮箱、电话等）
- 更改登录密码
- 添加个人简介和头像

用户可以通过顶部导航栏中的"个人资料"按钮访问个人资料管理页面。
